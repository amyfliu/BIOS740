# BIOS740 Final Project: Joint Biomedical NER and Relation Extraction

**Fang (Amy) Liu | Department of Biostatistics | UNC Chapel Hill**

This project applies [SpERT](https://github.com/lavis-nlp/spert) (Span-based Entity and Relation Transformer) with a SciBERT backbone to two biomedical knowledge graph datasets — ADKG (Alzheimer's Disease Knowledge Graph) and MDKG (Medical Disease Knowledge Graph) — for joint named entity recognition (NER) and relation extraction.

---

## Scripts

### Data Preparation
| Script | Description |
|---|---|
| `convert_to_spert.py` | Converts raw ADKG/MDKG JSON files to SpERT format. Maps character-level entity offsets to token indices and converts entity ID-based relation references to list indices. Generates `types.json` for each dataset. Outputs converted splits to `data/datasets/`. |
| `prepare_combined.py` | Combines the ADKG and MDKG train/dev splits into a single corpus for the transfer learning experiment. Generates a unified `combined_types.json` as the union of both type sets (11 entity types, 10 relation types). |

### Analysis
| Script | Description |
|---|---|
| `eda_spert.py` | Performs EDA on the raw ADKG and MDKG datasets. Prints a dataset overview table to terminal and saves four quality figures to `figures/`: entity distributions, relation distributions, top relation entity-type pairs, and entity/relation density comparison. |
| `edge_case_analysis.py` | Compares SpERT's test set predictions against gold labels to identify example edge cases. Outputs near-miss entities (correct type, span off by ≤1 token), missed entities by type, confused relation types, and failed long-range relations. Saves results to `logs/adkg_edge_cases.txt` and `logs/mdkg_edge_cases.txt`. |
| `entity_overlap_long_range.py` | Computes overlapping entity statistics and long-range dependency statistics (token distance between head and tail entities) across all splits of both datasets. |

### Training & Evaluation
| Script | Description |
|---|---|
| `spert.py` | Main SpERT entry point. Supports `train`, `eval`, and `predict` modes. Called with a `.conf` config file. From the original SpERT repository. |
| `run.sh` | Automates the full single-dataset pipeline: trains the model, extracts the saved model path from the log, updates the `best_model` symlink, and runs evaluation. Usage: `bash run.sh adkg` or `bash run.sh mdkg`. |
| `run_combined.sh` | Automates the combined dataset pipeline: trains on ADKG+MDKG combined, updates the symlink, then evaluates on ADKG and MDKG test sets separately. |

### SpERT Core (from original repo)
| Script | Description |
|---|---|
| `args.py` | Defines all command-line and config file arguments for SpERT. |
| `config_reader.py` | Parses `.conf` configuration files and passes arguments to the trainer. |

## Data

Original raw data files (`ADKG.json`, `MDKG.json`) are placed in the `spert/` root directory. Each file contains `train`, `dev`, and `test` splits in the following format:

```json
{
  "train": [
    {
      "doc_id": "...",
      "sent_id": "...",
      "text": "...",
      "entities": [{"id": "T1", "type": "disease", "start": 0, "end": 20, "text": "..."}],
      "relations": [{"type": "associated_with", "head": {"id": "T1", ...}, "tail": {"id": "T2", ...}}]
    }
  ]
}
```

Run `convert_to_spert.py` to convert these into SpERT's token-index format before training.

## Usage

### 1. Convert data
```bash
python convert_to_spert.py
```

### 2. Run EDA
```bash
python eda_spert.py | tee logs/eda_log.txt
```

### 3. Train and evaluate (single dataset)
```bash
bash run.sh adkg
bash run.sh mdkg
```

### 4. Train and evaluate (combined dataset)
```bash
python prepare_combined.py
bash run_combined.sh
```

### 5. Edge case analysis
```bash
python edge_case_analysis.py
```
